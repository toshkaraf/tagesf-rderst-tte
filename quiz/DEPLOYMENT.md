# Инструкция по развертыванию проекта

## Быстрый деплой на Vercel (рекомендуется)

### Шаг 1: Подготовка проекта

1. Убедитесь, что проект собирается для веба:
```bash
flutter build web --release
```

2. Создайте файл `vercel.json` в корне проекта (уже создан ниже)

### Шаг 2: Деплой на Vercel

1. Зарегистрируйтесь на [vercel.com](https://vercel.com) (бесплатно)
2. Установите Vercel CLI:
```bash
npm i -g vercel
```

3. Войдите в аккаунт:
```bash
vercel login
```

4. Деплой:
```bash
vercel --prod
```

Или используйте веб-интерфейс Vercel:
- Подключите GitHub репозиторий
- Укажите Build Command: `flutter build web --release`
- Укажите Output Directory: `build/web`
- Деплой произойдет автоматически

### Альтернатива: Netlify

1. Зарегистрируйтесь на [netlify.com](https://netlify.com)
2. Создайте файл `netlify.toml` в корне проекта
3. Подключите репозиторий через веб-интерфейс
4. Укажите:
   - Build command: `flutter build web --release`
   - Publish directory: `build/web`

## Настройка домена

### Бесплатный поддомен от Vercel/Netlify
- Vercel: `your-project.vercel.app`
- Netlify: `your-project.netlify.app`

### Покупка недорогого домена (рекомендуется)
- **Namecheap**: от $0.99/год для .xyz, от $8.88/год для .com
- **GoDaddy**: часто есть промо-предложения
- **IONOS**: хорошие цены для .de доменов (актуально для Германии)

После покупки домена подключите его в настройках Vercel/Netlify.

## Обновление контента

После изменений в проекте:
1. Закоммитьте изменения в Git
2. Vercel/Netlify автоматически пересоберут и задеплоят проект
3. Или запустите деплой вручную через CLI

## Хранение данных

Для загрузки презентаций и контента можно использовать:
- **GitHub**: храните файлы в репозитории
- **Firebase Storage**: бесплатный тариф (5 GB)
- **Cloudinary**: бесплатный тариф для изображений
- **GitHub Releases**: для больших файлов
